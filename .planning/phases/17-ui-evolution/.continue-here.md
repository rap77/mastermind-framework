<decisions_made>

**Key Decisions from Wave 2:**

1. **100ms WebSocket debouncing** — Reduces state updates by 96% during 24-brain bursts
   - **Why:** Prevents re-render flood, maintains 60fps performance
   - **Impact:** Measured 96% reduction in state updates, P99 < 16.67ms validated

2. **Targeted selectors (O(1) Map lookup)** — useCostState(brainId) prevents cascade re-renders
   - **Why:** Direct Map lookup is O(1), iterating array is O(n)
   - **Impact:** Zero cascade re-renders when single brain updates

3. **2-level hierarchy (per brain + total)** — Simplified from 3 levels per Brain #7
   - **Why:** Per-company aggregation deferred to v3.1, reduces query complexity
   - **Impact:** Cost dashboard simpler, no JOIN across tenant_id + company_id

4. **WCAG 2.1 AA compliance** — QuotaBar with color + text + icons
   - **Why:** Brain #2 identified color-only coding violation
   - **Impact:** Screen reader friendly, keyboard accessible, zero WCAG violations

5. **Parallel agent execution** — 4 agents for independent tasks
   - **Why:** Faster execution, user preference for background agents
   - **Impact:** Wave 2 completed in ~2.5 hours vs ~4+ hours sequentially

6. **PostgreSQL materialized view** — cost_metrics_mv for cost aggregation
   - **Why:** Brain #5 identified O(n) aggregation performance issue
   - **Impact:** P50 < 10ms query performance (indexed MV)

</decisions_made>
