## Objectives

Build the multi-tenant company switcher in the CompanyRail (left column) with:
- Draggable company ordering via @dnd-kit
- Visual status indicators per company (live agents, unread inbox)
- Company-specific branding/icons
- Sync across tabs via localStorage
- Tenant isolation (data separation per company)

## Success Criteria

1. CompanyRail displays all user's companies with visual status indicators
2. Companies can be reordered via drag-and-drop (persisted to localStorage)
3. Clicking a company switches the active tenant context
4. Company state syncs across browser tabs (localStorage events)
5. Tenant isolation enforced (API requests include tenant_id header)
6. Keyboard navigation works (Tab, Enter, Arrow keys)
7. Visual regression baseline captured before layout modifications

## Implementation Notes

**Drag-and-drop library choice:** @dnd-kit (not React Flow built-in)
- React Flow DnD is for graph nodes only, not list reordering
- @dnd-kit is lighter, more accessible, works with vertical lists

**localStorage sync strategy:**
- Use `storage` event listener for cross-tab sync
- Event fires when localStorage changes in another tab
- Rehydrate store on event trigger

**Tenant isolation enforcement:**
- Client-side: Always include `X-Tenant-ID` header
- Server-side: Validate tenant_id belongs to user (JWT sub claim)
- Fallback: Return 403 if tenant mismatch detected

**Status polling optimization:**
- Use WebSocket push in future (Phase 16 infrastructure)
- For now, poll every 30s (acceptable for MVP)
- Cache status in companyStore to reduce API calls

## Rollback Plan

If drag-and-drop causes performance issues:
1. Remove @dnd-kit dependency
2. Replace with simple move up/down buttons
3. Keep localStorage sync and tenant isolation

If tenant isolation breaks existing API:
1. Make `X-Tenant-ID` header optional (backward compatibility)
2. Add feature flag: `useTenantIsolation: boolean`
3. Gradual rollout per endpoint
