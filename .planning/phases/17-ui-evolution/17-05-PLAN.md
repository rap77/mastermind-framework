## Implementation Notes

**Fuzzy search library choice:**
- Option A: fuse.js (battle-tested, but adds dependency)
- Option B: Custom implementation (lighter, full control)
- **Recommendation:** Custom implementation (simple score-based matching)

**Keyboard shortcut implementation:**
- Use `useEffect` to add global listener for Cmd+K / Ctrl+K
- Prevent default browser behavior (Ctrl+K opens dev console in some browsers)
- Remove listener on component unmount (cleanup)

**Dialog component:**
- Use shadcn/ui Dialog (built on Radix UI)
- Focus trap: Tab cycles within dialog
- Escape key: Closes dialog (built-in)
- Click outside: Closes dialog (built-in)

**Virtualization:**
- If command list > 100 items, use react-window (virtualized list)
- For MVP (~50 commands), no virtualization needed
- Monitor performance, add virtualization if laggy

## Rollback Plan

If fuzzy search causes performance issues:
1. Replace with exact match search (startsWith)
2. Reduce debouncing to 100ms (faster feedback)
3. Limit results to 20 items (reduce DOM size)

If keyboard navigation conflicts:
1. Remove global listener, add button trigger only
2. Use different shortcut (Cmd+Shift+P instead of Cmd+K)
3. Add setting to disable/customize shortcut
