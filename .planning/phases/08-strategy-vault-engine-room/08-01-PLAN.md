Build new API key management route module with show-once pattern. **No Redis — DB-only soft-delete with revoked_at field.**

1. **Data model: APIKey**
   - id: UUID (primary key)
   - user_id: str (FK to user)
   - key_hash: str (bcrypt hash of full key)
   - prefix: str (first 8 chars for display)
   - suffix: str (last 4 chars for display)
   - created_at: datetime
   - last_used_at: datetime | null
   - revoked_at: datetime | null (null = active, non-null = revoked)
   - metadata: dict (optional, e.g. name="Claude API key")

2. **Endpoint: GET /api/keys**
   - Response: `{ keys: List[APIKeyMasked] }`
   - APIKeyMasked fields: id, prefix, suffix, created_at, last_used_at, metadata
   - Implementation:
     ```python
     user_id = get_current_user(request)
     keys = db.query(APIKey).filter(APIKey.user_id == user_id, APIKey.revoked_at == None).all()
     return { "keys": [mask_key(k) for k in keys] }
     ```
   - Auth: JWT required

3. **Endpoint: POST /api/keys**
   - Body: `{ name: str (optional) }`
   - Response: `{ id: str, full_key: str, prefix: str, suffix: str, message: "Save this key, you won't see it again" }`
   - Implementation:
     ```python
     user_id = get_current_user(request)
     full_key = "mmsk_" + secrets.token_hex(32)
     key_hash = bcrypt.hashpw(full_key.encode(), bcrypt.gensalt()).decode()
     db.add(APIKey(id=uuid.uuid4(), user_id=user_id, key_hash=key_hash,
                   prefix=full_key[:13], suffix=full_key[-4:], ...))
     db.commit()
     return { "id": ..., "full_key": full_key, "prefix": ..., "suffix": ..., "message": "..." }
     ```
   - Auth: JWT required
   - IMPORTANT: Full key shown ONCE in response only. Never logged. Frontend must copy immediately.

4. **Endpoint: DELETE /api/keys/{id}**
   - Path param: id (APIKey UUID)
   - Response: `{ status: "revoked" }`
   - Implementation:
     ```python
     user_id = get_current_user(request)
     key = db.get(APIKey, id)
     if not key or key.user_id != user_id: raise HTTPException(403)
     key.revoked_at = datetime.utcnow()
     db.commit()
     # No Redis — revoked_at field is the source of truth
     return { "status": "revoked" }
     ```
   - Auth: JWT required
   - Immediate revocation via revoked_at (no grace period, no cache layer)

5. **Key validation on API calls:**
   - Middleware: On request, extract `x-api-key` header
   - Validation:
     ```python
     raw_key = request.headers.get("x-api-key")
     # Query DB by prefix (first 13 chars) for O(1) candidate lookup
     candidate = db.query(APIKey).filter(
         APIKey.prefix == raw_key[:13],
         APIKey.revoked_at == None
     ).first()
     if not candidate or not bcrypt.checkpw(raw_key.encode(), candidate.key_hash.encode()):
         raise HTTPException(401)
     # Update last_used_at (non-blocking background task)
     background_tasks.add_task(update_last_used, candidate.id, db)
     ```
   - No Redis dependency — prefix-indexed DB query is fast enough for v2.1 load

6. **Rate limiting (Brain #7 gap B):**
   - Without Redis, bcrypt-per-request is vulnerable to spam → FastAPI event loop DoS
   - Use `slowapi` with in-memory store (sufficient for v2.1 internal tool)
   - Apply rate limit to API key validation middleware: max 60 requests/minute per IP
   - Pattern:
     ```python
     from slowapi import Limiter
     from slowapi.util import get_remote_address
     limiter = Limiter(key_func=get_remote_address)
     @router.get("/protected")
     @limiter.limit("60/minute")
     async def protected_route(request: Request): ...
     ```
   - **Auth latency guardrail:** Add test asserting bcrypt check < 200ms. If exceeded in v2.2, migrate to SHA-256 + salt or add LRU cache layer.

6. **Error cases:**
   - POST without auth → 401
   - DELETE non-existent key → 404
   - DELETE key belonging to other user → 403
