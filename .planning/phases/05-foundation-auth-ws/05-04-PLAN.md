## Gap Diagnosis

**Error reportado (UAT Test 10):**
```
Uncaught Error: [Immer] minified error nr: 0
Uncaught TypeError: Cannot add property 1, object is not extensible at Array.push
at Object.updateBrain
```

**Root cause:** Línea 28 en brainStore.ts
```typescript
get()._queue.push(brain)  // ❌ Mutación directa de estado congelado por Immer
```

**Problema:** Cuando usas `immer` middleware de Zustand, el estado retornado por `get()` está congelado (frozen) por Immer para garantizar inmutabilidad. Intentar mutarlo directamente (`push`, `set`, etc.) lanza error.

**Solución:** Todas las mutaciones deben ocurrir dentro del callback `set()`, donde Immer provee un `draft` mutable.

## Current Implementation Issues

```typescript
updateBrain: (brain) => {
  get()._queue.push(brain)         // ❌ Mutación directa de estado congelado
  if (!get()._rafId) {             // ❌ Lectura de estado congelado (ok, pero inconsistente)
    const id = requestAnimationFrame(() => {
      get()._drainQueue()          // ❌ Llamada a método que usa set() internamente
    })
    set(state => { state._rafId = id })
  }
}
```

## Expected Fix Pattern

```typescript
updateBrain: (brain) => {
  set(state => {
    state._queue.push(brain)       // ✅ Mutación dentro de set() con draft mutable
    if (!state._rafId) {           // ✅ Lectura de draft mutable
      const id = requestAnimationFrame(() => {
        get()._drainQueue()        // ✅ Ok: _drainQueue ya usa set() internamente
      })
      state._rafId = id            // ✅ Mutación dentro de set()
    }
  })
}
```

## Why This Works

1. **Immer + set() callback:** Cuando llamas `set(state => { ... })` con middleware immer, el parámetro `state` es un **draft mutable** producido por Immer usando proxies
2. **Mutaciones permitidas:** Dentro del callback, puedes hacer `push`, `set`, `splice`, etc. directamente
3. **Frozen get():** El estado retornado por `get()` es el **estado congelado actual**, solo para lectura
4. **RAF batching:** La lógica de acumular en _queue y drenar antes del paint se mantiene intacta, solo movemos la mutación dentro de set()
