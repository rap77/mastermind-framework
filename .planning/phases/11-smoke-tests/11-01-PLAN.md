   - Example: `./verify_feed_isolation.sh brain-04-frontend BRAIN-FEED-04-frontend.md`
   - Step 1: Pre-flight check — `git status --porcelain` must be clean. If dirty, warn and exit 3.
   - Step 2: `git stash --include-untracked` (WSL2 belt-and-suspenders per RESEARCH.md Pitfall 4)
   - Step 3: Print `READY: Dispatch $BRAIN_ID now. Press ENTER when agent completes.` and `read` for human input
   - Step 4: After ENTER, run `git diff --name-only` to get changed files
   - Step 5: Check for CRITICAL FAIL — if `BRAIN-FEED.md` (global, exact match) is in diff output: print `CRITICAL FAIL: Global BRAIN-FEED.md was modified`, `git stash pop`, exit 2
   - Step 6: Check for FAIL — filter changed files removing `.planning/$EXPECTED_FEED`. If remaining non-empty lines exist: print `FAIL: Unexpected files modified: $UNEXPECTED`, `git stash pop`, exit 1
   - Step 7: PASS — print `PASS: Only $EXPECTED_FEED modified (or no feeds modified — rejection-only)`, `git stash pop`, exit 0

   Key details:
   - Use `grep -v "^\.planning/$EXPECTED_FEED$"` for exact path matching (not substring)
   - The global feed check uses `grep -q "^\.planning/BRAIN-FEED\.md$"` (anchored regex, not substring — avoids matching `BRAIN-FEED-01-product.md`)
   - Add `set -euo pipefail` at top
   - Add `#!/bin/bash` shebang
   - Per Pitfall 1: "no feed modified at all" is valid for adversarial dispatches (agent correctly refused to write). Script should NOT fail in this case.

3. `chmod +x tests/smoke/verify_feed_isolation.sh`
