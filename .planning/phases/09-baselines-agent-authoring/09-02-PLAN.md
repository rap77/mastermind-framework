**## Stack Hard-Lock**
The ONLY approved stack. Any suggestion outside this list = Stack Hallucination = Rating 1.
- Framework: Next.js 16 (App Router, no Pages Router)
- UI: React 19 (Compiler DISABLED — conflicts with React.memo on RF nodes)
- Language: TypeScript strict mode (no implicit any, no explicit any without suppression comment)
- Styling: Tailwind 4 (CSS-only config — no tailwind.config.js)
- State: Zustand 5 + Immer (never Redux, never Context for performance-critical state)
- State access: useBrainState(id) targeted selector — never useStore() global
- Package manager: pnpm (Node.js), uv (Python 3.14)
- Never: npm, yarn, pip, poetry, conda
- Suggesting ANY library not in root uv.lock or pnpm-lock.yaml = Stack Hallucination = Rating 1

**## File Architecture**
Only create files in:
- apps/web/src/ — frontend code (screaming architecture: feature-first, not layer-first)
- apps/api/ — backend code
- .planning/ — project planning docs
Any file outside these locations requires explicit orchestrator approval before creation.

**## WebSocket Protocol**
Only use fields defined in the existing WS schema. Never invent protocol fields.
Key locked patterns:
- wsDispatcher is a module singleton
- Token obtained via /api/auth/token endpoint (Server Action reading httpOnly cookie)
- RAF batching lives in brainStore (not in wsDispatcher)
- 16ms drain cycle — 24 events max per frame

**## Cross-Domain Anti-Patterns**
These trigger rejection across ALL brain domains:
- Never functions > 50 lines (split if needed)
- Never `any` types without explicit suppression comment and reason
- Never commented-out production code
- Never hardcoded credentials — not even in test examples (Security Bypass = Rating 1)
- Never manual production access steps — direct DB changes, SSH without automation (Toil-Inducer = architecture failure)
- Never useStore() global selector — always useBrainState(id) or equivalent targeted selector
- NODE_TYPES at module level — never inline in JSX (prevents infinite re-render loop)

**## Feed Write Scope**
- Agents write ONLY to their own domain feed: .planning/BRAIN-FEED-NN-domain.md
- Global .planning/BRAIN-FEED.md is READ-ONLY for all agents
- Orchestrator writes to BRAIN-FEED.md after cross-domain synthesis
- A brain writing to BRAIN-FEED.md directly = context pollution = architectural violation

**## Oracle Pattern (for rejections)**
Valid rejection format:
"Rejected: [specific library or pattern] violates [constraint].
Source: global-protocol.md > [Section] | brain-NN-domain/warnings.md > [pattern name]"

Generic rejections without source citation = Rating 2 maximum. Cite the exact source.

**## Delta-Velocity Reminder**
- Rating 3 (Peer): correct, stack-aware, PR-ready with minor tweaks
- Rating 4 (Senior): detects non-obvious optimization or gap not in the ticket
- Rating 5 (Architect): proposes game-changing solution that unlocks the next roadmap phase
- T1 < 300 seconds (5 min) = agent is profitable vs manual skill
