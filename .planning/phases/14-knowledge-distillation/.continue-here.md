<decisions_made>

**Decisiones clave de arquitectura (lockeadas por Brain #7 92/100):**

1. **Quality Score 3-tier:** >=3.0 (template), >=1.0 (record), <1.0 (discard)
   - Fórmula: `(Precision × Success_Probability) / (T1 × Tokens) × 1000`
   - Penalties: Twaddle 50%, Inversion 30%

2. **Rejection Filter:** SQL `WHERE quality_score >= 1.0 AND status != 'rejected'`
   - Threshold: 1.0 previene "dead offers"
   - Filtrado mandatorio en retrieval

3. **TTL Ceiling:** 90-day expiry con `expires_at` column
   - Previene unbounded growth ceiling
   - P95 retrieval latency guardrail: >200ms → kill switch

4. **High-Value Detection:** 3 criterios
   - Duration > 5min (complejidad)
   - Planning score changed (pivot)
   - `mm:complete-phase` invocation

5. **Post-Session Hook:** FastAPI `BackgroundTasks.add_task()`
   - Non-blocking → Brain #7 evalúa en background
   - Usuario no espera evaluación

6. **Template Storage:** Tabla separada `knowledge_templates`
   - Screaming Architecture (templates ≠ experience logs)
   - Success rate + usage count tracking

7. **Dashboard Priority:** System health FIRST, outcome metrics SECOND
   - Health: record count, avg quality_score, rejection rate, P50/P90 latency, T1 trend
   - Outcome: delta-velocity, knowledge yield, planning accuracy

8. **Testing Strategy:** 31 tests nuevos, todos offline
   - 13 tests BLOCKER verification (scoring, rejection, TTL, high-value)
   - 18 tests adicionales (templates, analytics)
   - Total: 631 + 407 + 31 = 1069 tests

**Por qué estas decisiones:**
- Evitar statistical leakage (rechazos sin logging →学习中ad invisible)
- Prevenir unbounded growth (records acumulan sin límite → degradación)
- Filtrar ruido (dead offers poluyen memoria → outputs peores)
- Monitorear salud del sistema (no dashboard → muerte silenciosa)

</decisions_made>
