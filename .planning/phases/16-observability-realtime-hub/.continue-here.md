<decisions_made>

**Phase 16 Key Decisions:**

1. **In-memory Ghost Mode buffer** — Prevents PostgreSQL pool exhaustion during replay (thundering herd mitigation)

2. **Bounded WebSocket channels** — max_connections=2000, buffer=256 prevents OOM cascade

3. **Unary gRPC first** — Defer bi-directional streaming until metrics prove unary is bottleneck (Brain #7 Condition #3)

4. **Connection limit in WebSocketHub** — Mutex-protected check in hub.connect() NOT in handler (prevents race conditions)

5. **Specific SLIs over vague criteria** — All "works" converted to measurable thresholds (P95 < 500ms, etc.)

6. **Time buffers (20-50%)** — Added to all plans based on complexity (Rust async +50%, WebSocket +40%)

7. **Atomic commits per plan** — Each plan has its own commit for easy rollback

**Bug Fix Decision:**
- Removed race-prone check from websocket_handler
- Rely on WebSocketHub::connect() which locks mutex during check AND increment
- Added immediate connection close when connect() returns error
</decisions_made>
