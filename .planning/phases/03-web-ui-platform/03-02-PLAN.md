    1. **mastermind_cli/web/static/js/websocket.js**: WebSocket manager
       - WebSocketManager class:
         * connect(taskId, token): Open WebSocket connection
         * disconnect(): Close connection
         * on(event, callback): Register event listener
         * emit(event, data): Send message to server (optional)
         * reconnect(): Auto-reconnect with exponential backoff

       - Connection URL: `ws://localhost:8000/ws/tasks/${taskId}?token=${token}`
       - Event types: task_update_batch, task_complete, task_failed, error
       - Reconnection logic:
         * Exponential backoff: 1s, 2s, 4s, 8s, 16s (max)
         * Max retries: 5 (then give up)
         * On reconnect: Request resync from last_event_id

       - Throttling: Handle 300ms batch updates from server
       - Buffer: Store recent events for Ghost Mode (if disconnected <5min)

       - On task creation: Call wsManager.connect(taskId, token)
       - On task_update_batch event: Update tasks in list (batch update)
       - On task_complete event: Show success notification
       - On task_failed event: Show error notification
       - On disconnect: Show "Reconnecting..." status (Ghost Mode)
       - On reconnect: Resync task state from API

    3. **Real-time UI updates**:
       - Update task status in list
       - Update progress bar (if progress data available)
       - Show live token count (if streaming tokens)
       - Animate status changes (smooth transitions)
       - Auto-scroll to active task

    4. **Ghost Mode** (from CONTEXT):
       - UI desaturated when disconnected (opacity 0.6)
       - Counter: "Last sync: 2m ago"
       - On reconnect: Highlight changes ("Task #5 completed while you were away")
       - Overlay: "Reconnecting..." (non-blocking)

    5. **Fallback to polling** (from CONTEXT):
       - If WebSocket fails, switch to HTTP polling
       - Adaptive frequency: 2s when active, 10s when idle
       - Poll GET /api/tasks/{id} every N seconds
       - Stop polling when WebSocket reconnects

    6. **Error handling**:
       - Show user-friendly error on connection fail
       - Don't show raw WebSocket error codes
       - Log errors to console (for debugging)
       - Implement exponential backoff for retries

    Integration with Alpine.js:
    ```javascript
    function dashboard() {
      const wsManager = new WebSocketManager();

      return {
        tasks: [],
        init() {
          wsManager.on('task_update_batch', (data) => {
            // Batch update multiple tasks
            data.data.forEach(update => {
              const task = this.tasks.find(t => t.id === update.task_id);
              if (task) {
                task.status = update.status;
                task.progress = update.progress;
              }
            });
          });
        }
      };
    }
    ```
