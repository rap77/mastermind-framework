## Root Causes Diagnosed

1. **hash_token** used `bcrypt.gensalt()` every call → lookup by `WHERE key_hash = ?` always fails
   because stored hash and freshly-computed hash have different salts. Fix: SHA256 (deterministic).

2. **`:memory:` hardcoded everywhere in auth.py** → each request creates fresh empty DB.
   Login creates session → DB thrown away. Refresh looks in different empty DB → always 401.
   Fix: inject `get_db_path` dependency from shared `dependencies.py`.

3. **audit middleware** read `user_id` from `request.state` BEFORE `call_next` — auth
   dependencies set `request.state.user_id` INSIDE `call_next`. Always None.
   Fix: move read to AFTER `call_next`.

4. **get_db_path was local in tasks.py**, auth.py had none → tests needed to override both.
   Fix: shared `mastermind_cli/api/dependencies.py` with single `get_db_path`.
