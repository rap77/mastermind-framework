## Session 2026-03-16 - Integration Test Fixes

### Tests Fixed (Commit a8d95ec)

| # | Test | Issue | Fix |
|---|------|-------|-----|
| 1 | `test_concurrent_executions_no_crosstalk` | AssertionError on `result.brief` | Changed to `result.brief.problem_statement` (Brief is object, not string) |
| 2 | `test_legacy_brain_adapter_creates_isolated_context` | RuntimeError: 'MockLegacyBrain' object is not callable | Changed `brain_executor=MockLegacyBrain()` to `brain_executor=MockLegacyBrain().execute` |
| 3 | `test_web_auth_with_database` | TypeError: cannot unpack non-iterable coroutine object | Added `await` to `create_api_key()` call |
| 4 | `test_exponential_backoff_with_jitter` | assert 0 == 3 (call_times empty) | Changed from MagicMock to direct function, used thread-safe file-based call counting for `asyncio.to_thread` compatibility |

### Root Causes Identified

**Pattern 1: Brief object access**
- Problem: Tests treating `result.brief` as string
- Reality: `result.brief` is a `Brief` object with `problem_statement` attribute
- Fix: Access via `result.brief.problem_statement`

**Pattern 2: Callable vs Instance**
- Problem: `LegacyBrainAdapter` expects callable but receiving instance
- Fix: Pass `.execute` method, not the object itself

**Pattern 3: Async function not awaited**
- Problem: `create_api_key()` is async but called without await
- Fix: Add `await` keyword

**Pattern 4: asyncio.to_thread + MagicMock incompatibility**
- Problem: `MagicMock(side_effect=...)` doesn't work with `asyncio.to_thread`
- Fix: Use direct function, thread-safe counter via file I/O
