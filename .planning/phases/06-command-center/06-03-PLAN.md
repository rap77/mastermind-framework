# XSS Prevention Pattern (NEW - Security Gap Closure)
XSS (Cross-Site Scripting) is OWASP A03 - critical security risk:
- Stored XSS: Malicious brief stored in DB, displayed to other users
- Reflected XSS: Malicious brief reflected in response without sanitization

Solution: DOMPurify sanitization on BOTH client and server:
```typescript
import DOMPurify from 'dompurify';

// Client-side: Sanitize before sending to Server Action
const sanitizedBrief = DOMPurify.sanitize(brief, { ALLOWED_TAGS: [] });

// Server-side (FastAPI): Also sanitize before storage
from html import escape
brief_sanitized = escape(brief)  # Or use bleach library for HTML
```

ALLOWED_TAGS: [] = Only plain text allowed (no HTML, no Markdown for now)
Future: If Markdown support needed, use marked.js + DOMPurify combo
