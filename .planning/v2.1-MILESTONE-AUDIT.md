## Unsatisfied Requirements (Critical Gaps)

### ❌ ER-02 — API Key Management broken in production

**Description:** User can manage API keys (view masked, create, revoke)
**Root cause:** `apps/web/src/app/api/keys/route.ts` does not exist. `APIKeyManager.tsx` calls `fetch('/api/keys')` → Next.js 404. Config tab in Engine Room is completely broken.
**Fix:** Create Next.js proxy route `apps/web/src/app/api/keys/route.ts` (GET, POST, DELETE handlers) forwarding to FastAPI `/api/keys` with JWT from httpOnly cookie — same pattern as `/api/brains/route.ts`.

### ❌ ER-03 — BrainYAMLViewer orphaned

**Description:** User can view YAML config of any brain and copy to clipboard
**Root cause (a):** `BrainYAMLViewer.tsx` is never imported by any production file. No trigger point exists.
**Root cause (b):** `/api/brains/[id]/yaml` Next.js proxy route doesn't exist (`[id]/route.ts` proxies to `/api/brains/{id}` not `/yaml` suffix).
**Fix:** (a) Wire `BrainYAMLViewer` into `NodeDetailPanel` or Engine Room brain list with a "View YAML" button. (b) Create `/api/brains/[id]/yaml/route.ts` proxy.

### ❌ UX-01 — Focus Mode never activates

**Description:** User can enter Focus Mode — sidebar collapses, idle tiles dim, active elements highlighted
**Root cause:** `/nexus/page.tsx` renders `NexusCanvas` directly via `NexusCanvasLoader`, completely bypassing the `NexusPage` wrapper that implements `isFocusMode` CSS transitions and mounts `FocusModeBadge`. `orchestratorStore` state is set but never reaches the UI.
**Fix:** In `/nexus/page.tsx`, wrap the canvas render inside `<NexusPage>` component.

## Partial Requirements (Non-blocking)

### ⚠️ FND-03 — /command-center unprotected

**Root cause:** `/command-center` is at `apps/web/src/app/command-center/page.tsx` — outside the `(protected)` route group. `proxy.ts` is not at the `middleware.ts` path and is never invoked by Next.js runtime.
**Severity:** Security gap — Command Center accessible without JWT
**Fix option A:** Move `apps/web/src/app/command-center/` → `apps/web/src/app/(protected)/command-center/`
**Fix option B:** Rename `proxy.ts` → `middleware.ts` at the correct Next.js middleware location

### ⚠️ BE-02 — Task graph endpoint has no frontend consumer

**Root cause:** `GET /api/tasks/{id}/graph` exists and is BE-02 compliant. However `NexusCanvas` uses `fetchBrains()` for blueprint layout — the task graph is never fetched in production.
**Severity:** Tech debt — endpoint ready for v2.2 brain agent execution flows
**Note:** NEX-01, NEX-02, NEX-03 are all satisfied via the `fetchBrains()` path. BE-02 endpoint will be consumed in v2.2 when real task execution drives the graph.
