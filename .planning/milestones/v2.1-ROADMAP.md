## Milestone Summary

**Key Decisions:**

- Next.js 16 + React 19 over HTMX: production frontend with SSR data + Client real-time components
- React Flow over D3.js: better React integration, interactivity built-in, less custom code
- Zustand WS Dispatcher: single WS connection shared across all components (module singleton)
- Immer MapSet plugin: `enableMapSet()` required for Map<brainId, BrainState> in Immer set() callbacks
- RAF batching in brainStore (not WS handler): queues 24 events, drains before paint, 60fps
- NODE_TYPES/EDGE_TYPES at module level in NexusCanvas: prevents infinite re-render in React Flow
- dagreGraph module-level singleton with clearNode/clearEdge before each call: positional stability
- INSERT OR IGNORE for execution_writer concurrency: first writer wins, handles 24 simultaneous completions
- Separate api_keys_v2 table: avoids migrating legacy api_keys, no breaking changes
- react-virtuoso for O(1) memory virtual scroll: overscan=10 in Engine Room logs
- logFilterStore: manual JSON serialization for Set (avoids Zustand persist middleware Set issue)
- JWT at Server Components + Route Handlers (not only proxy.ts): CVE-2025-29927 mitigation
- React Flow CSS in globals.css @layer base: Tailwind 4 silently breaks handles/edges otherwise
- WS token handoff via /api/auth/token: server-side cookie read, token not in client bundle
- Next.js 16 middleware.ts → proxy.ts: cookies() async, CVE-2025-29927

**Technical Debt Deferred to v2.2:**

- WSBrainBridge disconnect() may race with CommandCenterWrapper WS in edge cases

- BrainMetadataSchema: REST brain responses now validated via Zod (wired in v2.1 tech debt cleanup)
