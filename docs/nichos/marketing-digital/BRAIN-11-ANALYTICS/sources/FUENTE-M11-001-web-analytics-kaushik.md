# FUENTE-M11-001: Web Analytics 2.0 (Avinash Kaushik)

## Tesis Central

> **"El 90% del valor de analytics no viene de las herramientas — viene de las personas que saben hacer las preguntas correctas. Contratar analistas brillantes vale 10 veces más que comprar software caro. Es el modelo 10/90: $10 en herramientas, $90 en cerebros."**

---

## 1. Principios Fundamentales

### El modelo 10/90 de inversión en analytics

Kaushik propone la regla de asignación de presupuesto analytics:

- **10%** en herramientas y tecnología
- **90%** en personas: analistas, formación, procesos

**Por qué:** Las herramientas sin interpretación son ruido. Google Analytics gratis + analista brillante > Tableau Enterprise + analista mediocre.

*Fuente: Web Analytics 2.0, Cap. 1 (Kaushik, 2009)*

### Los tres tipos de datos que necesitás

1. **Clickstream data:** Qué hacen los usuarios (Google Analytics, Adobe)
2. **Outcomes data:** Qué resultados se logran (conversiones, revenue, leads)
3. **Customer data:** Quiénes son los usuarios (CRM, demographics, surveys)

La mayoría de las empresas solo tiene el primero. Sin los tres, el análisis es incompleto.

*Fuente: Web Analytics 2.0, Cap. 2 (Kaushik, 2009)*

### Métricas vs. KPIs

- **Métricas:** Números que medís (pageviews, sessions, bounce rate)
- **KPIs:** Las métricas que importan para TU negocio y objetivo específico

El error clásico: medir todo sin saber qué importa. Un buen KPI tiene: objetivo de negocio vinculado, dueño, target, y frecuencia de revisión.

*Fuente: Web Analytics 2.0, Cap. 3 (Kaushik, 2009)*

## 2. Frameworks y Metodologías

### Framework: See-Think-Do-Care

El modelo de intención del cliente para segmentar analytics:

```
SEE: Audiencia más amplia con necesidad latente → Brand metrics, awareness
THINK: Audiencia considerando soluciones → Engagement, content metrics
DO: Audiencia lista para comprar → Conversion metrics, revenue
CARE: Clientes existentes → Retention, loyalty metrics
```

Cada etapa requiere KPIs diferentes. Mezclarlos genera confusión.

*Fuente: Kaushik, "The See-Think-Do-Care Framework" (2013, kaushik.net)*

### Framework: The Digital Marketing and Measurement Model (DMMM)

5 pasos para crear un plan de medición:

1. **Business Objectives:** ¿Qué intenta lograr el negocio?
2. **Goals for each objective:** ¿Qué significa el éxito?
3. **KPIs for each goal:** ¿Cómo medimos el éxito?
4. **Segments to analyze:** ¿Qué segmentos importan?
5. **Targets:** ¿Qué número define éxito vs. fracaso?

*Fuente: Kaushik, "DMMM" (kaushik.net, 2012)*

### Framework: Trinity de Análisis

Los tres tipos de análisis que hay que hacer siempre:

| Tipo | Pregunta | Herramienta |
|------|----------|-------------|
| **Behavior** | ¿Qué hacen los usuarios? | Google Analytics, Hotjar |
| **Experience** | ¿Qué piensan los usuarios? | Surveys, user testing |
| **Outcomes** | ¿Qué logra el negocio? | CRM, revenue dashboard |

*Fuente: Web Analytics 2.0, Cap. 5 (Kaushik, 2009)*

## 3. Modelos Mentales

### "Bounce Rate no es una métrica mala — depende del contexto"

El bounce rate tiene diferentes significados según el tipo de página:
- Blog post: 80% bounce es normal (leyó y se fue satisfecho)
- Landing page de producto: 80% bounce es un problema
- Página de contacto: 90% bounce puede ser perfecto (encontró el teléfono)

**Implicación:** Nunca interpretar una métrica sin contexto del objetivo de la página.

*Fuente: Web Analytics 2.0, Cap. 6 (Kaushik, 2009)*

### "All data in aggregate is crap"

Los promedios esconden insights. El dato agregado (ej: "el conversion rate es 2%") no dice nada. La segmentación (por canal, dispositivo, geografía, fuente) es donde está el verdadero insight.

*Fuente: Web Analytics 2.0, Cap. 4 (Kaushik, 2009)*

## 4. Criterios de Decisión

### Cuándo usar métricas de vanidad vs. métricas accionables

- **Métricas de vanidad** (pageviews, followers): Útiles para comunicar a stakeholders, no para tomar decisiones
- **Métricas accionables** (conversion rate por segmento, revenue per visitor): Únicas válidas para decisiones de inversión

**Regla:** Si la métrica no cambia una decisión, no gastar tiempo analizándola.

*Fuente: Web Analytics 2.0, Cap. 3 (Kaushik, 2009)*

### Cuándo invertir en A/B testing vs. análisis cualitativo

- **A/B testing:** Cuando hay suficiente tráfico (>1000 conversiones/mes por variante) y quiero optimizar
- **Cualitativo (surveys, user testing):** Cuando el tráfico es bajo o cuando no sé QUÉ testear

*Fuente: Web Analytics 2.0, Cap. 9 (Kaushik, 2009)*

---

## 5. Anti-patrones

### Anti-patrón: HiPPO-driven decisions

HiPPO = Highest Paid Person's Opinion. Tomar decisiones basadas en la opinión del CEO/CMO en lugar de datos es el anti-patrón más costoso en marketing digital.

*Fuente: Web Analytics 2.0, Cap. 1 (Kaushik, 2009)*

### Anti-patrón: Reportes sin recomendación

Un reporte de analytics que solo muestra números sin insights y recomendaciones de acción es un ejercicio de presentación, no de análisis.

*Fuente: Web Analytics 2.0, Cap. 11 (Kaushik, 2009)*

### Anti-patrón: Implementación de tracking sin plan de medición

Instalar GA sin definir primero qué KPIs importan genera data dumps inutilizables. Siempre: plan de medición primero, implementación después.

*Fuente: Web Analytics 2.0, Cap. 2 (Kaushik, 2009)*
