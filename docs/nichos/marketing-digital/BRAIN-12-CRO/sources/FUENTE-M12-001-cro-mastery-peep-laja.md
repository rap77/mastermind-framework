# FUENTE-M12-001: CRO Mastery (Peep Laja / CXL)

## Tesis Central

> **"El CRO sin investigación es adivinar. La mayoría de los equipos de CRO fallan porque testean hipótesis basadas en intuición, no en datos cualitativos y cuantitativos. El 80% del trabajo de CRO debe ser investigación. Solo el 20% es testing."**

---

## 1. Principios Fundamentales

### El problema con el CRO sin sistema

Las empresas que hacen CRO sin metodología cometen los mismos errores:

1. Testear cambios cosméticos (color de botones) sin evidencia de que el color sea el problema
2. Declarar victorias con muestras estadísticamente insuficientes
3. Implementar "mejoras" que aumentan conversión pero dañan otras métricas
4. No documentar qué se aprendió ni por qué funcionó

**Resultado:** Ciclos de optimización que no acumulan conocimiento y no generan mejora sostenida.

*Fuente: Laja, "CRO Foundation" (CXL, 2022)*

### Las tres preguntas de CRO

Antes de cualquier test, responder:

1. **¿Qué está pasando?** (Analytics: dónde se va la gente, qué páginas tienen problemas)
2. **¿Por qué está pasando?** (Investigación cualitativa: qué piensan los usuarios)
3. **¿Qué deberíamos cambiar?** (Hipótesis basadas en evidencia)

*Fuente: Laja, "CRO Research Framework" (CXL, 2022)*

## 2. Frameworks y Metodologías

### Framework: ResearchXL

El proceso de investigación más completo para CRO:

```
PASO 1: ANÁLISIS HEURÍSTICO
└── Evaluar el sitio contra principios de UX/CRO sin datos
└── Identificar problemas obvios de usabilidad, claridad, fricción

PASO 2: ANÁLISIS DE ANALYTICS
└── Identificar páginas con alta tasa de salida
└── Mapear el funnel con % de abandono por etapa
└── Segmentar por dispositivo, canal, geografía

PASO 3: ANÁLISIS TÉCNICO
└── Velocidad de carga por dispositivo
└── Errores en consola, broken links, formularios que fallan
└── Cross-browser/cross-device testing

PASO 4: ANÁLISIS DE MOUSE TRACKING (Hotjar/FullStory)
└── Heatmaps: dónde hacen click y dónde no
└── Scrollmaps: hasta dónde leen
└── Session recordings: comportamiento real de usuarios reales

PASO 5: INVESTIGACIÓN CUALITATIVA
└── Encuestas on-site (¿qué te impide comprar hoy?)
└── User testing (5 usuarios revelan 80% de los problemas)
└── Entrevistas con clientes (¿qué los motivó a comprar?)

PASO 6: ANÁLISIS DE MENSAJES
└── ¿El copy refleja el lenguaje que usan los clientes?
└── ¿Las objeciones clave están resueltas en la página?
└── ¿El value proposition es claro en 5 segundos?
```

*Fuente: Laja, "ResearchXL Methodology" (CXL, 2022)*

### Framework: PIE para priorización de tests

Priorizar qué testear primero:

```
PIE Score = (Potential + Importance + Ease) / 3

Potential (1-10): ¿Cuánto puede mejorar si se arregla?
Importance (1-10): ¿Cuánto tráfico/revenue pasa por esta página?
Ease (1-10): ¿Qué tan fácil es implementar el test?
```

**Ejemplo:**
- Página de checkout: P=9, I=10, E=6 → PIE Score = 8.3 (alta prioridad)
- Página de blog: P=4, I=3, E=9 → PIE Score = 5.3 (baja prioridad)

*Fuente: Laja, "Test Prioritization" (CXL, 2022)*

### Framework: The Conversion Trinity

Para que una página convierta, debe cumplir las tres condiciones:

1. **Relevance:** ¿La página coincide con la expectativa del usuario que llegó?
2. **Value:** ¿El valor es claro y suficiente para justificar la acción?
3. **Call to Action:** ¿Hay una acción clara y la fricción para tomarla es mínima?

Si alguna de las tres falla, la conversión falla independientemente de las otras dos.

*Fuente: Laja, "Conversion Trinity" (CXL, 2022)*

## 3. Modelos Mentales

### "El mejor test es el que no necesitás hacer"

Muchos problemas de conversión son obvios con investigación básica. Si el user testing muestra que el 4 de 5 usuarios no entiende el value proposition, no necesitás un A/B test para saber que hay que mejorar el copy.

*Fuente: Laja, "Research vs Testing" (CXL, 2022)*

### "Conversion rate es una métrica de resultado, no de optimización"

Optimizar directamente el conversion rate puede llevar a manipulaciones (dark patterns) que aumentan la métrica pero destruyen el negocio. Optimizar la experiencia del usuario que lleva a conversión es la estrategia sostenible.

*Fuente: Laja, "CRO Ethics" (CXL, 2022)*

## 4. Criterios de Decisión

### Cuánto tráfico necesitás para A/B testing

Regla mínima de Laja:
- Landing page: Mínimo 1,000 visitantes/mes por variante para tests en 2 semanas
- E-commerce checkout: Mínimo 500 transacciones/mes por variante
- Por debajo de estos umbrales: Priorizar investigación cualitativa, no tests

*Fuente: Laja, "Sample Size Requirements" (CXL, 2022)*

### Qué hacer cuando hay poco tráfico

Con < 500 conversiones/mes:
1. User testing (5-8 usuarios) → Insights de alta calidad sin necesitar volumen
2. Encuestas de salida → Entender por qué se van
3. Entrevistas con clientes → Entender por qué compraron
4. Análisis de mouse tracking → Ver comportamiento real

*Fuente: Laja, "Low Traffic CRO" (CXL, 2022)*

## 5. Anti-patrones

### Anti-patrón: Cambiar el color del botón primero

El mito del "botón rojo vs. verde" es el símbolo del CRO sin metodología. El color del botón raramente es el problema. El problema suele ser el value proposition, la fricción, o la confianza.

*Fuente: Laja, "CRO Myths" (CXL, 2022)*

### Anti-patrón: Declarar victoria demasiado rápido

Terminar un test cuando llega a 95% de significancia estadística, sin haber completado el tiempo planificado, infla la tasa de falsos positivos. La regla: completar el test independientemente de los resultados intermedios.

*Fuente: Laja, "Statistical Significance in CRO" (CXL, 2022)*

### Anti-patrón: Optimizar páginas de baja importancia

Mejorar el conversion rate de una página de blog con 100 visitas/mes tiene impacto casi cero. Siempre atacar las páginas con mayor volumen de tráfico en los pasos más críticos del funnel.

*Fuente: Laja, "CRO Prioritization" (CXL, 2022)*
