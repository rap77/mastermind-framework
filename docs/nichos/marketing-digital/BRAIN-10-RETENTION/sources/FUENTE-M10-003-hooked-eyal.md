# FUENTE-M10-003: Hooked (Nir Eyal)

## Tesis Central

> **"Las empresas que forman hábitos en sus usuarios tienen ventajas competitivas enormes. El Hook Model es el framework para diseñar productos que los usuarios no solo usan — sino que necesitan."**

---

## 1. Principios Fundamentales

### El Hook Model: 4 fases del hábito

```
TRIGGER → ACTION → VARIABLE REWARD → INVESTMENT
    ↑                                      |
    └──────────────── Loop ─────────────────┘
```

Cada ciclo del hook refuerza el hábito. Cuantos más ciclos, más profundo el hábito.

*Fuente: Hooked, Cap. 1 (Eyal, 2014)*

### Triggers: externos vs. internos

- **Triggers externos:** Notificaciones push, emails, SMS, ads — lo que usamos en marketing
- **Triggers internos:** Emociones, pensamientos, situaciones que activan el producto sin prompt externo

**El objetivo final:** Pasar de triggers externos (costosos, dependientes de nosotros) a triggers internos (el usuario piensa en el producto automáticamente ante ciertos contextos).

*Fuente: Hooked, Cap. 2 (Eyal, 2014)*

### Variable Rewards: por qué el hábito se forma

El cerebro se activa más ante recompensas **impredecibles** que predecibles. Tres tipos:

1. **Rewards of the Tribe:** Recompensas sociales (likes, comentarios, validación)
2. **Rewards of the Hunt:** Búsqueda de recursos (scroll infinito, search results)
3. **Rewards of the Self:** Dominio y progreso personal (gamification, streaks)

*Fuente: Hooked, Cap. 5 (Eyal, 2014)*

## 2. Frameworks y Metodologías

### Framework: Frequency vs. Utility Matrix

Eyal clasifica productos según dos ejes:

```
        Alta Frecuencia de Uso
               |
  Baja         |          Alta
  Utilidad  VITAMINS  | PAINKILLERS
             ________|________
              WASTE   | VITAMINS
  Alta         |       (with habit)
  Utilidad     |
               Baja Frecuencia
```

**Insight clave:** Los "vitamins" (nice to have) pueden convertirse en "painkillers" (necesidad) si forman hábito.

*Fuente: Hooked, Cap. 1 (Eyal, 2014)*

### Framework: Investment & Loaded Value

La fase Investment es lo que separa los productos con hábito de los que no lo tienen:

- **Data stored:** Historial, preferencias, contenido creado
- **Followers/connections:** Redes sociales construidas dentro del producto
- **Reputation:** Reviews, badges, puntos de confianza
- **Skills learned:** Tiempo invertido en aprender el producto

Cada inversión hace más costoso el abandono.

*Fuente: Hooked, Cap. 6 (Eyal, 2014)*

### Aplicación en Push Notifications

El Hook Model aplicado a push/email:

| Fase | Aplicación |
|------|-----------|
| **Trigger** | Push/email personalizado en el momento correcto |
| **Action** | CTA simple, fricción mínima, deep link directo al valor |
| **Variable Reward** | Contenido personalizado, sorpresa, novedad |
| **Investment** | Pregunta de feedback, personalización, configuración |

*Fuente: Hooked, Cap. 3-4 (Eyal, 2014)*

## 3. Modelos Mentales

### "Motivación + Habilidad + Trigger = Comportamiento"

(Adaptado del Fogg Behavior Model que Eyal cita)

Para que un usuario actúe ante un trigger:
- Debe estar suficientemente motivado (desire)
- Debe tener la capacidad de actuar (simplicity)
- El trigger debe llegar en el momento oportuno

Si el comportamiento no ocurre, ajustar primero la habilidad (simplificar), luego el trigger (timing), por último la motivación.

*Fuente: Hooked, Cap. 3 (Eyal, 2014)*

### "El primer usuario sos vos mismo"

Eyal propone el test del "fabricante + usuario":

- ¿Usarías tu propio producto?
- ¿Recomendarías tu propio producto?

Si la respuesta es no, estás construyendo manipulation, no engagement genuino.

*Fuente: Hooked, Conclusión (Eyal, 2014)*

## 4. Criterios de Decisión

### Cuándo usar push vs. email para el trigger

| Canal | Usar cuando | Evitar cuando |
|-------|-------------|---------------|
| **Push** | Acción time-sensitive, usuario activo en mobile | Contenido informativo largo, usuario inactivo >30 días |
| **Email** | Nurturing, contenido rico, decisiones de compra | Urgencia real, usuarios mobile-first |
| **SMS** | Urgencia alta, confirmaciones, ofertas flash | Contenido educativo, audiencias B2B formales |
| **In-app** | Usuario ya está en el producto | Usuarios que no abren la app |

*Fuente: Hooked, Cap. 2 (Eyal, 2014) + aplicación práctica*

## 5. Anti-patrones

### Anti-patrón: Triggers sin relevancia

Enviar notificaciones masivas sin segmentación destruye el hábito en lugar de crearlo. El usuario ignora o desactiva las notificaciones.

**Regla:** Menos triggers, más relevantes, mejor timing > Más triggers, genéricos, mal timing.

*Fuente: Hooked, Cap. 2 (Eyal, 2014)*

### Anti-patrón: Manipulación vs. persuasión

Eyal distingue:
- **Persuasión:** El producto entrega valor real que coincide con el Desired Outcome del usuario
- **Manipulación:** El producto explota vulnerabilidades psicológicas sin valor real

El primer modelo es sostenible. El segundo genera reputación negativa y regulación.

*Fuente: Hooked, Conclusión (Eyal, 2014)*

### Anti-patrón: Variable rewards predecibles

Si el usuario puede predecir exactamente qué recibirá, la recompensa pierde poder. Mantener variabilidad genuina (no artificial) es clave.

*Fuente: Hooked, Cap. 5 (Eyal, 2014)*
