# FUENTE-156: Community Events — Eventbrite Team

## Tesis Central

> **"Events are the heartbeat of community. Online interaction builds relationships, but events accelerate those relationships by 10x. The most successful communities have regular, predictable events that members can count on."**

---

## 1. Principios Fundamentales

> **P1: Consistency Beats One-Off Spectaculars**
- A monthly meetup that always happens builds more community than one amazing annual conference.
> *Fuente: [Community Event Strategy (Eventbrite, 2023)]*
> *Contexto: When designing community event calendar.*

> **P2: Events Serve Different Community Purposes**
- Some events are for learning, some for networking, some for celebration. Know why you're hosting each event.
> *Fuente: [Event Types Framework (Eventbrite, 2023)]*
> *Contexto: When planning event mix and calendar.*

> **P3: Virtual Events Still Have Value Post-Pandemic**
- Virtual events aren't "second best"—they enable access, convenience, and global reach that in-person can't.
> *Fuente: [Hybrid Event Strategy (Eventbrite, 2023)]*
> *Contexto: When deciding between virtual, in-person, or hybrid formats.*

> **P4: The Before and After Matter More Than the Event**
- Community building happens in the anticipation before and the processing after, not just during the event.
> *Fuente: [Event Lifecycle (Eventbrite, 2023)]*
> *Contexto: When designing event communications and follow-up.*

> **P5: Events Should Create Peer Connections, Not Just Content Delivery**
- The most valuable part of community events isn't the stage content—it's the connections members make with each other.
> *Fuente: [Event Design (Eventbrite, 2023)]*
> *Contexto: When structuring event agenda and networking opportunities.*

## 2. Frameworks y Metodologías

### Framework 1: The Community Event Mix

**Fuente:** [Community Event Calendar (Eventbrite, 2023)]
**Propósito:** Design balanced event calendar that serves different member needs
**Cuándo usar:** When planning community programming and events

**The 5 Event Types:**

**1. Learning Events (Education)**
- **Format:** Workshops, webinars, classes, tutorials
- **Frequency:** Weekly or bi-weekly
- **Goal:** Skill development, knowledge sharing
- **Examples:** "How to Use X Feature," "Industry Expert Q&A"
- **Engagement:** Q&A, polls, breakout discussions
- **Best for:** Product communities, professional communities

**2. Networking Events (Connection)**
- **Format:** Matched 1:1s, speed networking, small groups
- **Frequency:** Weekly or monthly
- **Goal:** Peer connections, relationship building
- **Examples:** "Member Matchups," "Coffee Chats," "Office Hours"
- **Engagement:** Pre-matched pairs, icebreakers, conversation prompts
- **Best for:** All community types (core activity)

**3. Social Events (Belonging)**
- **Format:** Happy hours, game nights, casual hangouts
- **Frequency:** Monthly
- **Goal:** Fun, belonging, informal connection
- **Examples:** "Trivia Night," "Virtual Happy Hour," "Game Night"
- **Engagement:** Low-pressure activities, fun, casual
- **Best for:** Member bonding, community culture

**4. Celebration Events (Recognition)**
- **Format:** Awards, showcases, anniversaries, milestones
- **Frequency:** Quarterly or annual
- **Goal:** Recognize members, mark progress, create rituals
- **Examples:** "Member of the Month," "Annual Awards," "Community Birthday"
- **Engagement:** Member spotlights, awards, storytelling
- **Best for:** Culture, motivation, community identity

**5. Collaborative Events (Co-Creation)**
- **Format:** Hackathons, design sprints, working sessions
- **Frequency:** Quarterly
- **Goal:** Create together, contribute to mission
- **Examples:** "Community Hackathon," "Feature Co-Design," "Content Sprint"
- **Engagement:** Hands-on work, small teams, deliverables
- **Best for:** Innovation, product feedback, advocacy

**Sample Monthly Calendar:**
- **Week 1:** Learning Event (workshop)
- **Week 2:** Networking Event (1:1 matches)
- **Week 3:** Social Event (game night)
- **Week 4:** Rotating (Celebration quarterly, Collaborative quarterly)

**Output esperado:** Balanced event calendar keeping community engaged across different member needs

### Framework 2: The Virtual Event Experience Framework

**Fuente:** [Virtual Event Best Practices (Eventbrite, 2023)]
**Propósito:** Design engaging virtual events that don't feel like Zoom fatigue
**Cuándo usar:** When planning virtual community events

**Pre-Event (Build Anticipation):**
1. **Teaser Content:** Preview what's coming, guest bios, topics
2. **Event Page:** Detailed agenda, speaker bios, what to expect
3. **Reminder Sequence:** 7 days, 1 day, 1 hour before event
4. **Discussion Starter:** Prompt in community: "What questions do you have for our speaker?"
5. **Icebreaker Prep:** Share icebreaker question in advance

**During Event (Engagement Design):**
1. **Opening Energy:** Start with energy—music, warm host, quick icebreaker (5 min)
2. **Interaction Every 10 Minutes:** Poll, chat question, breakout, Q&A—never go 10 min without interaction
3. **Breakout Rooms:** For networking, small groups, discussions (5-8 people max)
4. **Chat Management:** Dedicated moderator monitoring chat, highlighting great questions
5. **Visual Variety:** Slides, screen sharing, videos, demos—change visual every 5-10 min
6. **Timebox Rigorously:** Start on time, end on time (respect attendees' time)

**Post-Event (Extend Value):**
1. **Immediate Follow-Up:** Thank you email within 24 hours with recording + key takeaways
2. **Community Discussion:** Thread in community to continue conversation
3. **Action Items:** Any commitments made during event
4. **Next Event Teaser:** Preview next event to maintain momentum
5. **Feedback Loop:** Survey attendees, share what you learned

**Virtual Event Platforms:**
- **Simple:** Zoom, Google Meet (up to 100)
- **Medium:** Hopin, Remo, Run the World (100-500)
- **Large:** Cvent, vFairs (500+)
- **Community Integration:** Circle, Discord (ongoing events)

**Output esperado:** Virtual events that are engaging, not exhausting

**Fuente:** [Event Experience Design (Eventbrite, 2023)]
**El modelo:** Events have an engagement arc: anticipation (before) → peak energy (during) → processing and integration (after). Most event planners focus only on "during," missing the opportunities before and after.
**Por qué funciona:** Explains why some events create lasting community impact while others are forgotten

## 4. Criterios de Decisión

### Trade-off: Virtual vs In-Person vs Hybrid

**Fuente:** [Event Format Decision (Eventbrite, 2023)]

**Opción A: Virtual-Only**
- **Ventajas:** Global access, lower cost, convenience, data capture
- **Desventajas:** Screen fatigue, less connection energy, harder networking
- **Mejor para:** Regular events, global communities, learning events

**Opción B: In-Person-Only**
- **Ventajas:** Highest energy, best connection, memorable experiences
- **Desventajas:** Geographic limitation, higher cost, logistical complexity
- **Mejor para:** Major annual events, local communities, celebration events

**Opción C: Hybrid**
- **Ventajas:** Best of both worlds, flexibility
- **Desventajas:** Most complex, expensive, risk of two unequal experiences
- **Mejor para:** Major events where some want in-person, others virtual

**Recomendación del experto:** Virtual for regular events (monthly/quarterly). In-person for annual flagship. Hybrid only when you have resources to do both experiences well. Poor hybrid is worse than choosing one and doing it excellently.

## 5. Anti-patrones

### Anti-patrón 1: One-Way Content Delivery

**Qué es:** Events that are just presentations with no interaction
**Consecuencias:** Low engagement, drop-off, members stop attending
**Qué hacer en su lugar:** Every 10 minutes, add interaction—polls, breakout, chat questions
