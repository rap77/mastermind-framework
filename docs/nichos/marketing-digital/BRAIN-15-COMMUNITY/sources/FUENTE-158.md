# FUENTE-158: Discord & Reddit Communities — Platform Teams

## Tesis Central

> **"Each platform has its own native culture. Discord communities feel different from Slack, which feel different from Reddit, which feel different from Facebook Groups. Build with the platform's native culture, not against it."**

---

## 1. Principios Fundamentales

### Discord Platform Principles

> **D1: Discord is About Real-Time Belonging**
- Discord thrives on spontaneous, real-time conversation. It's more like a virtual hangout than a message board.
> *Contexto: When designing Discord community structure and culture.*

> **D2: Voice Channels are Discord's Superpower**
- Voice channels create presence like being in the same room. They enable working together, hanging out, serendipitous connection.
> *Contexto: When leveraging Discord's unique features for community.*

> **D3: Roles and Permissions Enable Community Governance**
- Discord's role system lets members self-select into interests and access levels. Use this for segmentation and autonomy.
> *Contexto: When structuring Discord server roles and permissions.*

> **D4: Bots Extend Discord's Functionality**
- Discord bots can automate moderation, gamification, notifications, and custom features. Bots make Discord infinitely customizable.
> *Contexto: When enhancing Discord server with automation and engagement tools.*

### Reddit Platform Principles

> **R1: Reddit is About Asynchronous Discussion**
- Reddit excels at slow-burn, threaded discussion where quality rises to the top through voting.
> *Contexto: When designing Reddit community and content strategy.*

> **R2: Subreddits are Topic-Based, Not Brand-Based**
- Successful subreddits focus on topics/interests, not specific brands. r/Apple succeeds because it's about Apple, not because Apple runs it.
> *Contexto: When launching or managing subreddit communities.*

> **R3: Reddit Culture Values Authenticity and Skepticism**
- Redditors smell marketing from miles away. Authentic contribution to discussions works; self-promotion is downvoted to oblivion.
> *Contexto: When participating in Reddit as a brand or community manager.*

> **R4: Moderation is Critical on Reddit**
- Subreddit quality depends entirely on active moderation. Without it, spam, low-quality posts, and toxicity take over.
> *Contexto: When structuring Reddit moderation and community governance.*

## 2. Frameworks y Metodologías

### Framework 1: Discord Server Structure

**Fuente:** [Discord Community Guidelines (Discord, 2023)]
**Propósito:** Structure Discord server for engagement and scalability
**Cuándo usar:** When setting up or reorganizing Discord community

**Essential Channels:**

**1. Welcome Channel (#welcome)**
- **Purpose:** Onboarding, rules, introductions
- **Content:** Server rules, getting started guide, introduce-yourself thread
- **Settings:** Read-only for members, only moderators can post

**2** **General Channels (#general, #off-topic)**
- **Purpose:** Casual conversation, hangout space
- **Content:** Open discussion, anything goes within rules
- **Settings:** Open for all members, high activity

**3. **Announcements Channel (#announcements)**
- **Purpose:** Official updates, product news, events
- **Content:** Company announcements, feature releases, event notifications
- **Settings:** Read-only for members, only team can post

**4. **Help/Support Channel (#help)**
- **Purpose:** Member support, questions, troubleshooting
- **Content:** Member questions, community answers, team support
- **Settings:** Open for all, support team monitors

**5. **Voice Channels (Voice, Gaming, Study, etc.)**
- **Purpose:** Real-time voice hangouts
- **Content:** Members dropping in to chat, work together, hang out
- **Settings:** Open for all, persistent channels (always available)

**Optional but Recommended:**

**Introductions Channel (#introductions)**
- Dedicated space for member introductions
- Template for introductions (name, location, interests, etc.)
- Helps members learn about each other

**Roles Channel (#roles)**
- Members self-assign roles (by region, interests, expertise)
- Uses reaction roles or bot commands
- Enables segmentation and personalization

**Resources Channel (#resources)**
- Curated helpful resources, guides, FAQs
- Organized by topic
- Reduces repetitive questions

**Events Channel (#events)**
- Community events, AMAs, scheduled activities
- Event calendar, signups, reminders
- Central hub for community programming

**Organization Best Practices:**
- **Categories:** Group related channels in categories (e.g., "General," "Support," "Events")
- **Channel Limits:** Don't create too many channels—better to have active channels than dead channels
- **Clear Naming:** Use channel names that clearly indicate purpose
- **Channel Order:** Most important channels at top, least used at bottom

**Roles Structure:**

**Default Roles:**
- **@everyone:** All members (use sparingly for pings)
- **@member:** Standard member role (basic access)
- **@moderator:** Community moderators (additional permissions)
- **@admin:** Server admins (full control)

**Custom Roles:**
- **@verified:** Verified community members (optional verification)
- **@contributor:** Active contributors, content creators
- **@alumni:** Long-time members, founders
- **Regional:** By location (e.g., @north-america, @emea)
- **Interests:** By topic (e.g., @developers, @marketers, @designers)

**Role Permissions:**
- **Basic Members:** Read/write in general channels, read announcements, access voice
- **Moderators:** All member permissions + moderation tools, manage channels
- **Admins:** All permissions + server management, role management

**Output esperado:** Well-organized Discord server that's easy to navigate and scales

### Framework 2: Subreddit Strategy and Management

**Fuente:** [Reddit Community Guide (Reddit, 2023)]
**Propósito:** Build and manage successful subreddit communities
**Cuándo usar:** When creating or growing subreddit for brand or topic

**Subreddit Launch:**

**1. Define Subreddit Purpose**
- **Topic Focus:** What is this subreddit about? (Be specific)
- **Target Audience:** Who is this for?
- **Differentiation:** How is this different from existing subreddits?
- **Value Prop:** Why would someone subscribe and participate?

**2. Create Subreddit**
- **Name:** Short, memorable, indicates topic (r/ProductName, r/Topic)
- **Description:** Clear explanation of subreddit purpose and rules
- **Sidebar:** Important links, rules, community info
- **Banner:** Visual identity, subreddit branding

**3. Establish Rules and Guidelines**
- **Post Rules:** What types of posts are allowed/encouraged?
- **Behavior Rules:** How should members interact? (be respectful, no spam, etc.)
- **Format Guidelines:** Any specific formatting requirements?
- **Moderation Policy:** How are rules enforced?

**4. Initial Content Seeding**
- **Starter Posts:** Create 10-20 high-quality posts to set tone
- **Discussion:** Start conversations, ask questions, share insights
- **Resources:** Post helpful resources, guides, FAQs
- **Consistency:** Post regularly to show activity

**5. Promotion and Growth**
- **Cross-Promotion:** Share with other relevant subreddits (carefully—no spam)
- **Own Channels:** Promote to email list, social media, existing community
- **Influencers:** Partner with Reddit influencers in your niche
- **Paid:** Reddit ads (optional, for initial boost)

**Ongoing Management:**

**Content Moderation:**
- **Review Queue:** Check new posts queue regularly
- **Approve/Remove:** Approve on-topic posts, remove off-topic/spam
- **Flair:** Use post flair to categorize content
- **Lock Discussions:** Lock threads that have run their course or become problematic

**Community Engagement:**
- **Discussion Posts:** Regular discussion prompts ("What's everyone working on?")
- **AMAs:** Host AMAs with experts, team members, community figures
- **Events:** Regular events (weekly challenges, monthly themes)
- **Recognition:** Recognize quality contributors, highlight great posts

**Moderation Team:**
- **Recruit Moderators:** From active, positive community members
- **Mod Guidelines:** Clear moderation guidelines and expectations
- **Mod Coordination:** Private mod subreddit or Discord for coordination
- **Tools:** Use moderation bots (AutoModerator, Toolbox)

**Metrics:**
- **Subscribers:** Growth rate
- **Active Users:** Daily/weekly active users
- **Posts/Comments:** Volume and quality of content
- **Engagement:** Upvotes, comments per post
- **Reports:** Spam reports, rule violations

**Output esperado:** Thriving subreddit with active participation and quality content

## 3. Modelos Mentales

### Modelo Mental 1: Platform Culture Fit

**Fuente:** [Platform Native Culture (Discord/Reddit, 2023)]
**El modelo:** Each platform has a native culture. Discord = casual, real-time, voice-first. Reddit = asynchronous, discussion-focused, skeptical of marketing. Communities that match platform culture thrive; those that don't struggle.
**Por qué funciona:** Explains why the same community works well on Discord but fails on Reddit
**Cuándo usarlo:** When choosing platforms and designing community strategy for each

## 4. Criterios de Decisión

### Trade-off: Discord vs Reddit vs Slack vs Circle

**Fuente:** [Platform Selection Guide (Discord/Reddit, 2023)]

**Discord:**
- **Best for:** Real-time chat, gaming, tech, younger demographics
- **Pros:** Free, voice channels, rich features, huge existing user base
- **Cons:** Message fatigue, search challenges, noisy
- **Use when:** You want casual, real-time community

**Reddit:**
- **Best for:** Topic discussions, news, content curation
- **Pros:** Huge audience, SEO, voting surfaces quality
- **Cons:** Skeptical of brands, requires constant moderation
- **Use when:** You want discussion around a specific topic

**Slack:**
- **Best for:** Professional communities, work-focused
- **Pros:** Professional feel, great search, thread support
- **Cons:** Expensive, message limits, feels like work
- **Use when:** Professional, work-oriented community

**Circle:**
- **Best for:** Membership communities, courses + community
- **Pros:** Ownership, monetization, beautiful UX
- **Cons:** Cost, smaller existing user base
- **Use when:** Paid membership community

**Recommendation:** Start where your audience already is. For tech/gaming → Discord. For B2B professional → Slack. For consumer discussion → Reddit. For membership → Circle.

## 5. Anti-patrones

### Anti-patrón 1: Cross-Posting Identical Content Across Platforms

**Qué es:** Posting same content to Discord, Reddit, Slack, etc. without adaptation
**Consecuencias:** Doesn't work, feels lazy, community rejects it
**Qué hacer en su lugar:** Adapt content for each platform's culture and format
