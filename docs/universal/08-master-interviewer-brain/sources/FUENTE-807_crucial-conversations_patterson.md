## Conocimiento Destilado

### 1. Principios Fundamentales

> **P1: Start with Heart**
> Antes de hablar, clarificar lo que realmente querés. ¿Querés aprender? ¿Persuadir? ¿Resolver?
> *Contexto: En entrevistas difíciles, primero define tu objetivo real.*

> **P2: Learn to Look**
> Prestá atención a cuándo la conversación se vuelve crucial (seguridad amenazada).
> *Contexto: Detectar cuando el usuario se cierra o se pone a la defensiva.*

> **P3: Make It Safe**
- Cuando hay seguridad, la gente puede decir cualquier cosa
- Cuando no hay seguridad, el diálogo se rompe
> *Contexto: Crear un environment donde el usuario se sienta seguro para ser honesto.*

> **P4: Master My Stories**
- Entre lo que pasa y cómo reaccionamos, hay una historia que nos contamos
- Controlar esa historia controla la reacción
> *Contexto: Cuando el usuario dice algo difícil, no reaccionar inmediatamente. Revisar tu historia interna.*

### 2. Frameworks y Metodologías

#### FM1: The STATE Framework

**Propósito:** Hablar sobre temas difíciles sin ofender.

| Letra | Significado | Acción |
|-------|-------------|--------|
| **S** | Share your facts | Compartir hechos objetivos, no juicios |
| **T** | Tell your story | Expresar tu interpretación como tal, no como hecho |
| **A** | Ask for others' paths | Invitar a la otra persona a compartir su perspectiva |
| **T** | Talk tentatively | Usar lenguaje tentativo ("Podría ser que...", "Me parece que...") |
| **E** | Encourage testing | Invitar desafíos ("¿Qué estoy pasando por alto?", "¿Qué pensás?") |

**Ejemplo en entrevista:**
- ❌ "Tu producto es terrible"
- ✅ "Noté que 3 de mis 5 intentos fallaron (facts). Me parece que hay un problema de estabilidad (story). ¿Qué ves vos? (ask)"

#### FM2: Build Safety

**Propósito:** Crear un environment seguro para el diálogo abierto.

**Dos condiciones de seguridad:**

1. **Mutual Purpose** — "¿Estamos trabajando hacia el mismo objetivo?"
   - Clarificar intenciones
   - Asegurar que ambos quieren el mismo resultado

2. **Mutual Respect** — "¿Me respetás y te respeto?"
   - Mostrar respeto genuino
   - No invalidar emociones o perspectives

**Cómo restablecer seguridad:**
- "No quiero que pensemos que tengo todas las respuestas"
- "Mi objetivo es entender tu perspectiva"
- "Valor tu honestidad"

#### FM3: Listen to Understand

**Propósito:** Escuchar para entender, no para responder.

**Técnicas:**
- **Ask for elaboration:** "¿Podés contarme más sobre eso?"
- **Paraphrase:** "Lo que estoy escuchando es..." (verify understanding)
- **Mirror feelings:** "Parece que esto te frustra..."
- **Acknowledge:** "Entiendo por qué pensarías eso"

### 3. Modelos Mentales

#### MM1: The Pool of Shared Meaning

**Concepto:** Cuando todos aportan al diálogo, se crea un "pool" de significado compartido.

**En entrevistas:**
- Más contributions = mejor información
- Safe environment = más contributions
- Unsafe = people withhold information

**Objetivo:** Maximizar el pool de shared meaning.

#### MM2: Silence ≠ Agreement

**Problema:** Interpretar silencio como acuerdo.

**Realidad:**
- Silencio puede ser miedo, desacuerdo, o confusión
- No asumas que "todo está bien"

**Solución:**
- "¿Qué pensás sobre esto?"
- "¿Hay algo que no dijiste que te gustaría agregar?"

#### MM3: The Fool's Choice

**Falso dilema:** "O soy honesto y lastimo a la persona" o "Miento y soy amable"

**Third option:** Ser honesto Y respetuoso simultáneamente.

**En entrevistas:**
- No es "tienes razón" o "estás equivocado"
- Es: "Entiendo tu perspectiva. Te comparto lo que veo..."

### 4. Criterios de Decisión

#### CD1: Detecting Unsafe Conversations

| **Señal de unsafe** | **Ejemplo** |
|---------------------|-------------|
| Usuario se calla repentinamente | Deja de hablar, respuestas cortas |
| Usuario se defiende | "No es mi culpa", "No dije eso" |
| Usuario ataca | "Vos no entendés nada", "Esto es una pérdida de tiempo" |
| Usuario se emociona | Frustración visible, enojo |
| Usuario desconecta | Mira el reloj, mira para otro lado |

**Qué hacer:** Detener el tema actual y restablecer seguridad.

#### CD2: When to Use Each Technique

| Situación | Técnica |
|-----------|---------|
| Usuario se cierra | Build safety (Mutual Purpose + Respect) |
| Usuario emocional | Label feelings + Mirror emotions |
| Usuario en desacuerdo | Ask for others' paths + Encourage testing |
| Usuario agresivo | Maintain safety, no counter-attack |
| Usuaio callado | Ask for perspective + Tentative language |

#### CD3: AMPP in Interviews

**Ask, Mirror, Paraphrase, Prime:**

1. **Ask:** "¿Qué pensás sobre esto?"
2. **Mirror:** "Parece que algo te preocupa..."
3. **Paraphrase:** "Entiendo que decís..."
4. **Prime:** "Podría ser que..." (dar una possible interpretation para iniciar diálogo)

### 5. Anti-patrones

#### AP1: Jumping to Conclusions

**Mal:** Interpretar rápido juzgar
- "Este usuario solo quiere quejarse"
- "No está interesado"

**Bien:** Ask + Mirror + Paraphrase
- "¿Qué te preocupa?"
- "Parece que esto es importante para vos"
- "Lo que estoy escuchando es..."

#### AP2: Ad Homine

**Mal:** Atacar a la persona
- "Vos sos demasiado negativo"
- "No entendés el contexto"

**Bien:** Separar persona del problema
- "Entiendo que esto es frustrante"
- "Hablemos del problema, no de quién tiene la culpa"

#### AP3: Withholding Information

**Mal:** No compartir información por "seguridad"
- "No puedo decirte por qué..."
- "Es una decisión de negocio"

**Bien:** Compartir contexto
- "La razón detrás de esto es..."
- "Te comparto el contexto completo..."

- [ ] Start with Heart: ¿Qué quiero realmente?

- [ ] Ask + Mirror + Paraphrase + Prime
- [ ] Master your stories (no reaccionar inmediatamente)
- [ ] Escuchar para entender, no para responder

### Para Mantener el Diálogo
- [ ] Mutual Purpose: "Queremos lo mismo"
- [ ] Mutual Respect: "Valor tu perspectiva"
- [ ] Encourage testing: "¿Qué estoy pasando por alto?"
- [ ] Talk tentatively: "Podría ser que...", no "Es obvio que..."
