# Pattern: Gap Closure Workflow

**Pattern:** After phase execution, always verify → plan gaps → execute gaps → complete phase.

**Workflow:**
1. Execute all original plans (`/mm:execute-phase N`)
2. Run verification (`/mm:complete-phase N` or gsd-verifier)
3. If gaps found: `/mm:plan-phase N --gaps`
4. Execute gap closure plans (`/mm:execute-phase N`)
5. Complete phase (`/mm:complete-phase N`)

**Why:** Plans executed in isolation often miss integration layers. Verification catches these gaps before calling phase "done."

**Example:** Phase 18 executed 7 plans (webhook receiver, DLQ, adapters, UI). Verification found 15 gaps: webhook route not registered, gRPC bridge missing, queue depth stubbed. Gap closure plans created to fix.

**Key Files:**
- `NN-VERIFICATION.md` - Generated by verifier, lists all gaps with status
- `NN-PLAN.md` (gap_closure: true) - Plans to fix gaps
- `.continue-here.md` - Handoff file for session resumption
