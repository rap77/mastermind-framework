## Patterns Discovered

### 1. Async/Await Pattern (3 instances)
**Problem:** Tests calling async functions without `@pytest.mark.asyncio` decorator and `await` keyword.

**Example Fix:**
```python
# Before (WRONG)
def test_create_api_key(self):
    full_key, response = create_api_key(create_data)

# After (CORRECT)
@pytest.mark.asyncio
async def test_create_api_key(self):
    full_key, response = await create_api_key(create_data)
```

**Files affected:** `tests/unit/test_auth_api_keys.py`

### 2. Missing Task Creation Pattern (2 instances)
**Problem:** Tests calling `execute_brain()` without creating task in database first.

**Example Fix:**
```python
# Before (WRONG)
executor = ParallelExecutor(...)
result = await executor.execute_brain(task_id="test-001", ...)

# After (CORRECT)
executor = ParallelExecutor(...)
await task_repo.create(task_id="test-001", brain_id="brain-01")  # ADD THIS
result = await executor.execute_brain(task_id="test-001", ...)
```

**Files affected:** `tests/unit/test_error_formatter.py`

### 3. Mock Async/Sync Mismatch Pattern (3 instances)
**Problem:** Tests creating async mocks (`async def query_brain`) when real code calls sync methods (`def call_mcp`).

**Example Fix:**
```python
# Before (WRONG) - Mock is async
class MockMCPClient:
    async def query_brain(self, brain_id: str, query: str):
        await asyncio.sleep(0.1)
        return {...}

# After (CORRECT) - Mock is sync (matches real interface)
class MockMCPClient:
    def call_mcp(self, brain_id: str, query: str):
        time.sleep(0.1)
        return MCPResponse(brain_id=brain_id, response="...", success=True)
```

**Key Insight:** The real TypeSafeMCPWrapper has sync methods. If mock is async, it doesn't match the interface and tests fail silently or with cryptic errors.

**Files affected:** `tests/integration/test_parallel_execution.py`, `tests/integration/test_pure_function_arch.py`

### 4. TaskGroup Result Collection Pattern
**Problem:** Awaiting tasks inside TaskGroup context serializes execution instead of parallel.

**Example Fix:**
```python
# Before (WRONG) - Serializes execution
async with asyncio.TaskGroup() as tg:
    tasks = {tg.create_task(f()) for f in functions}
    for brain_id, task in tasks.items():
        result = await task  # BAD: Awaits one by one
        results[brain_id] = result

# After (CORRECT) - Gets results after all complete
async with asyncio.TaskGroup() as tg:
    tasks = {tg.create_task(f()) for f in functions}
# TaskGroup already awaited all tasks
for brain_id, task in tasks.items():
    results[brain_id] = task.result()  # GOOD: Just get result
```

**Files affected:** `mastermind_cli/orchestrator/task_executor.py`

### 5. Sync I/O Blocking Event Loop Pattern
**Problem:** Calling sync methods inside async functions blocks the event loop, preventing true parallelism.

**Example Fix:**
```python
# Before (WRONG) - Blocks event loop
async def _call_brain(self, brain_id: str, query: str):
    response = self.mcp_client.call_mcp(brain_id, query)  # SYNC CALL
    return {...}

# After (CORRECT) - Runs in thread pool
async def _call_brain(self, brain_id: str, query: str):
    response = await asyncio.to_thread(self.mcp_client.call_mcp, brain_id, query)
    return {...}
```

**Impact:** Speedup improved from 0.16x (slower!) to 4.59x (faster!) for parallel execution.

**Files affected:** `mastermind_cli/orchestrator/task_executor.py`
